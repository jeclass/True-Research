# Head-to-head: True Research vs hosted deep research

The True Research side is done — both runs finished 2026-07-02 (see EVIDENCE.md
for stats; full run folders live on the operator's machine). To complete the
comparison:

## 1. Collect competitor outputs

Paste each prompt file below, **verbatim**, into the competitor's deep-research
mode and let it run to completion:

- `prompt-glp1.txt` → ChatGPT Deep Research, and/or Gemini Deep Research
- `prompt-local-llms.txt` → same
- `prompt-prediction-markets.txt` → any hosted deep-research product (Claude,
  ChatGPT, Gemini) — this one was run on True Research from a FRESH public
  clone (v1.2), i.e. the exact experience a new user gets.

Save each output (export or copy the full report, citations included) into this
folder as:

```
chatgpt-glp1.md                gemini-glp1.md
chatgpt-local-llms.md          gemini-local-llms.md
claude-prediction-markets.md   gemini-prediction-markets.md
```

## 2. Blind scoring

With the files in place, run the judge (a fresh Opus session scoring the §8
rubric; judge spend goes to a separate ledger so it never contaminates run
economics — see `run_evals.py`). Scoring dimensions, applied to anonymized
reports:

1. **Citation accuracy** — do cited sources actually support the claim they're
   attached to? (Spot-check sample per report.)
2. **Verification** — are load-bearing claims independently checked, and does
   the report distinguish verified from unverified?
3. **Depth** — coverage of the prompt's enumerated sub-questions.
4. **Source quality** — primary sources vs aggregators/news summaries, per the
   prompts' explicit requirements.

## 3. Publish

Results (scores + methodology + the outputs) get a section in EVIDENCE.md.
True Research's own reports are not committed to the repo because they embed
large amounts of third-party page text; figures are read from each run's
`ledger.json`/`run.json`, and the reports can be regenerated by anyone with
the one-command invocations documented in EVIDENCE.md.
